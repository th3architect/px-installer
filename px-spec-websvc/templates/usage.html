Portworx configurator form for Kubernetes

ValueDescription
REQUIRED PARAMETERS
cluster: Specifies the unique name for the Portworx cluster. (example: test_cluster)
kvdb: Points to your key value database, such as an etcd cluster or a consul cluster. (example: etcd:http(s)://etcd.fake.net:2379)
OPTIONAL PARAMETERS
type: Select Portworx deployment type.
drives: Specify comma-separated list of drives. (example: /dev/sdb,/dev/sdc)
diface: Specifies the data interface. This is useful if your instances have non-standard network interfaces. (example: eth1)
miface: Specifies the management interface. This is useful if your instances have non-standard network interfaces. (example: eth1)
coreos: REQUIRED if target nodes are running coreos. (example: true)
master: If true, PX will run on the master node. For Kubernetes 1.6.4 and prior, this needs to be true (default is false) (example: true)
zeroStorage: Instructs PX to run in zero storage mode on kubernetes master. (example: true)
force: Instructs PX to use any available, unused and unmounted drives or partitions. PX will never use a drive or partition that is mounted. (example: true)
KVDB CONFIGURATION PARAMETERS
etcdPasswd: Username and password for ETCD authentication in the form user:password (example: username:password)
etcdCa: Location of CA file for ETCD authentication. (example: /path/to/server.ca)
etcdCert: Location of certificate for ETCD authentication. (example: /path/to/server.crt)
etcdKey: Location of certificate key for ETCD authentication. (example: /path/to/server.key)
acltoken: ACL token value used for Consul authentication. (example: 398073a8-5091-4d9c-871a-bbbeb030d1f6)
LIGHTHOUSE CONFIGURATION PARAMETERS
token: Portworx lighthouse token for cluster. (example: a980f3a8-5091-4d9c-871a-cbbeb030d1e6)
env: Comma-separated list of environment variables that will be exported to portworx. (example: API_SERVER=http://lighthouse-new.portworx.com)
